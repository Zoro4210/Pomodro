# 🧘‍♂️ Pomodro — The Zen-Focused Multi-Branch Project

Welcome to the official repository for **Pomodro**, a premium deep-work ecosystem designed to shield you from distractions and elevate your productivity.

This project is organized into two primary branches to separate the application logic from the interactive documentation.

---

## 🏗️ Repository Branch Structure

To maintain a clean and efficient development workflow, this repository is split into two specialized branches:

### 📱 `app` Branch (Production Application)
The **`app`** branch is the core of the ecosystem. It contains the complete source code for the Pomodro Flutter application. 

- **Contents**: 
  - `lib/`: All Dart source code for the focus timer, orientation logic, and state management.
  - `android/`, `ios/`, `web/`: Platform-specific configurations and build files.
  - `pubspec.yaml`: Project dependencies and environment settings.
  - `assets/`: App-specific icons, fonts, and local storage assets.
  - `zenith_crimson/`: The bespoke design system documentation and specifications.

- **Primary Goal**: Building and deploying the Pomodro mobile experience.

### 📖 `manual` Branch (Interactive Site)
The **`manual`** branch hosts the premium interactive manual—a high-fidelity website dedicated to guiding users through the Pomodro philosophy and usage.

- **Contents**:
  - `index.html`: The core structure of the immersive manual landing page.
  - `css/`: Custom Vanilla CSS styling, featuring glassmorphism and modern layouts.
  - `js/`: Interactive logic, including orientation demos and scroll-reveal triggers.
  - `assets/`: High-resolution images, video demos, and the project's signature cat mascot.

- **Primary Goal**: Providing a stunning, accessible web interface for documentation and onboarding.

---

## 🚀 How to Navigate

Depending on what you are looking for, you can switch between these branches on GitHub:

1. **For the App Source Code**: Switch to the **`app`** branch.
2. **For the Manual Website Source**: Switch to the **`manual`** branch.

> [!TIP]
> If you are a user looking for the final experience, you can download the latest **APK** from the `app` branch's releases or view the hosted **Manual Site** (if deployed via GitHub Pages from the `manual` branch).
